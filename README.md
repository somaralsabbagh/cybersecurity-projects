<h2 id="malware-image-classification">Malware Image Classification</h2>

<h3>Problem</h3>
<p>Conventional malware detection pipelines degrade under family variation, packing, and obfuscation. The target was a robust classifier that captures structural binary patterns through visual encoding.</p>

<h3>Approach</h3>
<ul>
  <li>Converted executable binaries into image representations.</li>
  <li>Applied preprocessing quality controls and dataset consistency checks.</li>
  <li>Benchmarked multiple transfer-learning backbones.</li>
  <li>Organized train/evaluate/inference into a structured workflow.</li>
  <li>Standardized outputs for reproducibility and technical reporting.</li>
</ul>

<h3>Results</h3>
<ul>
  <li>Stable high-performance classification across selected architectures.</li>
  <li>Clear model-selection signal through comparative metrics.</li>
  <li>Reproducible artifact generation for review and validation.</li>
</ul>

<details>
  <summary><strong>Architecture Notes</strong></summary>
  <ul>
    <li>Notebook-heavy workflow transformed into modular project structure.</li>
    <li>Separation between configuration, training logic, and reporting outputs.</li>
    <li>Reproducibility treated as a core engineering requirement.</li>
  </ul>
</details>

<h2 id="sdn-hybrid-aaa-security-platform">SDN Hybrid AAA Security Platform</h2>

<h3>Problem</h3>
<p>Single-layer authentication is inadequate for segmented SDN environments requiring identity assurance and policy-aware access control.</p>

<h3>Approach</h3>
<ul>
  <li>Implemented <strong>hybrid AAA</strong>: RADIUS NAC + Kerberos step-up verification.</li>
  <li>Added controller-level enforcement for role, session, quarantine, and rate controls.</li>
  <li>Integrated DHCP snooping and ARP inspection for L2 trust hardening.</li>
  <li>Exposed telemetry through API and dashboard views.</li>
  <li>Structured phased execution for repeatable security validation.</li>
</ul>

<h3>Results</h3>
<ul>
  <li>Delivered practical defense-in-depth architecture for campus-style SDN.</li>
  <li>Validated progression from baseline connectivity to hybrid enforcement.</li>
  <li>Produced operational evidence via metrics, events, and service status signals.</li>
</ul>

<details>
  <summary><strong>Architecture Notes</strong></summary>
  <ul>
    <li>Layered controls across admission, proof-of-identity, and runtime policy.</li>
    <li>Clear boundary between orchestration scripts and controller security logic.</li>
    <li>Operational observability implemented as a first-class system capability.</li>
  </ul>
</details>
